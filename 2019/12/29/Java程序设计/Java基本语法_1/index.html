1、Java 平台 与 历史版本

    1. Java 平台
    • Java SE 标准版 - Java Standard Edition（J2SE,Java 2 Platform Standard Edition）
      • 桌面应用
    • Java EE 企业版 - Java Enterprise Edition（J2EE,Java 2 Platform Enterprise Edition）
      • 网络应用（Web 应用）
    • Java ME 微型版 Java Mirco Edition（J2ME,Java 2 Platform Mirco Edition）
      • 手机应用
    1. Java 历史版本
    • Java 1.0 -1995
    • Java 1.2 -1998 Java 2
    • Java 1.3 -改进
    • Java 1.4 -稳定
    • Java 1.5（Java 5）- 增加语法
    • Java 1.6（Java 6）- 广泛
    • Oracle并购Sun
    • Java 1.7（Java 7） 2011
    • Java 1.8（Java 8） 2014

  • 3.Java 推动力

    • JCP (Java Community Process) Java 社区过程
    • JSR (Java Specification Requests) Java 规范
      • 不同规范不一样
      • eg：JSR 335
        • Lambda Expressions for the Java Programming Language（Lambda 表达式）
      • 这是 Java 8 的最重要的新特性。

    1. 高性能
    1. 安全性

3、Java 运行机制

  1. Java 三种核心机制

    • Java 虚拟机 -JVM（Java virtual Machine）
    • 代码安全性检测（Code Security）
    • 垃圾收集机制（Garbage Collection）

  2. Java 程序的编译与运行

    • Java 源程序 （.java) -编译
      • javac 编译
    • 字节码（bytecode） （.class) -运行 【平台无关】
      • java 运行 调用JVM
      • .class 经过 运行 才可以得到 cpu指令

  3. Java运行环境

  • JRE（The Java Runtime Environment) Java运行环境
  • JRE = JVM + API (Lib)
  • JRE 运行程序时的三项主要功能:
    • 加载代码：由 class loader 完成
    • 校验代码：由 bytecode verifier 完成
    • 执行代码：由 runtime interpreter 完成
  • Java 运行环境 是由虚拟机来装载这些程序并且调用相应的指令来具体的执行

Q1：为什么Java是跨平台的？
A：字节码文件与平台无关，字节码文件放到不同的系统，它都有不同的虚拟机来执行。

Q2：为什么Java是安全的？
A：Java 运行程序时 JRE中有校验代码的过程，由 bytecode verifier 完成。

  1. Java自动垃圾回收技术
  • 系统级线程跟踪存储空间的分配情况。
  • 在JVM的空闲时，检查并释放那些可被释放的存储空间。
  • 程序员无须也无法精确控制和干预该回收过程。
  1. Java JDK
  • JDK = JRE + Tools
  • JRE = JVM + API

    要开发 Java 程序，要下载 JDK；
    如果只运行 Java 程序 ，则下载 JRE；

Java JDK 提供的工具：

  1. Java 编译器 - javac.exe
  2. Java 执行器 - java.exe
  3. 文档生产器 - javadoc.exe
  4. Java 打包器 - jar.exe
  5. Java 调试器 - jdb.exe

4、面向对象程序设计

  • 类与对象的关系

    • 类是对象的抽象/模板
    • 对象是类的实例
      类和对象有时候统称为对象,精确称呼：后者为对象实例。

  • 面向对象的三大特征

    • 封装性
      • 模块化：将属性和行为封装在类中，
        程序定义很多类。
      • 信息隐蔽：将类的细节隐藏起来，
        用户只能通过受保护的接口访问某个类。
    • 继承性
      • 父类和子类之间共享数据和方法
        优点：
        • 更好地抽象与分类
        • 增加代码的重用率
        • 提高可维护性
    • 多态性
      • 不同对象受到同一个消息（调用方法）可产生不同的效果
      • 实现细节则由接受对象自行决定

        什么是面向对象程序设计思想的要点？

        1. 认为客观世界都是由对象组成的

        2. 程序分析和设计都是围绕着
          有哪些对象类
          每个类有哪些属性和方法
          类之间的关系（继承、关联等）
          对象之间发送消息（调用方法）
