Git&Github_Git 命令介绍、名词解释

The text is as follows :

Install

Configure tooling

git config --global user.name "[name]"

Sets the name you want attached to your commit transactions

设置要附加到提交事务的名称

git config --global user.email "[email address]"

Sets the email you want attached to your commit transactions

设置要附加到提交事务的电子邮件

git config --global color.ui auto

Enables helpful colorization of command line output

启用命令行输出的着色

Branches

git branch [branch-name]

Creates a new branch

创建新分支

git checkout [branch-name]

Switches to the specified branch and updates the
working directory

切换到指定的分支并更新工作目录

git merge [branch]

Combines the specified branch’s history into the
current branch. This is usually done in pull requests,
but is an important Git operation.

将指定分支的历史记录合并到当前分支。这通常是在拉取请求中完成的，但是是一个重要的Git操作。

git branch -d [branch-name]

Deletes the specified branch

删除指定的分支

Create repositories

git init

Turn an existing directory into a git repository

将现有目录转换为Git存储库

git clone [url]

Clone (download) a repository that already exists on GitHub, including all of the files, branches, and commits

克隆（下载）一个已经存在于Github上的存储库，包括所有的文件、分支和提交

The .gitgnore file

Sometimes it may be a good idea to exclude files from being
tracked with Git. This is typically done in a special file named
.gitignore . You can find helpful templates for .gitignore
files at github.com/github/gitignore.

Synchronize changes

git fetch

Downloads all history from the remote tracking branches

从远程跟踪分支下载所有历史记录

git merge

Combines remote tracking branch into current local branch

将远程跟踪分支合并到当前本地分支

git push

Uploads all local branch commits to GitHub

将所有本地分支提交上载到GitHub

git pull

Updates your current local working branch with all new
commits from the corresponding remote branch on GitHub.
git pull is a combination of git fetch and git merge

使用所有新的从GitHub上相应的远程分支提交。git pull是git fetch和git merge的组合

Make changes

git log

Lists version history for the current branch

列出当前分支的版本历史记录

git log --follow [file]

Lists version history for a file, including renames

列出文件的版本历史记录，包括重命名

git diff [first-branch]...[second-branch]
Shows content differences between two branches

显示两个分支之间的内容差异

git show [commit]
Outputs metadata and content changes of the specified commit

输出指定提交的元数据和内容更改

git add [file]
Snapshots the file in preparation for versioning

快速文件以备版本控制

git commit -m "[descriptive message]"
Records file snapshots permanently in version history

在版本历史记录中永久记录文件快照

Redo commits

git reset [commit]
Undoes all commits after [commit], preserving changes locally

撤消[commit]之后的所有提交，在本地保留更改

git reset --hard [commit]
Discards all history and changes back to the specified commit

放弃所有历史记录和对指定提交的更改

Glossary

**
git: an open source, distributed version-control system
GitHub: a platform for hosting and collaborating on Git repositories
commit: a Git object, a snapshot of your entire repository compressed into a SHA
branch: a lightweight movable pointer to a commit
clone: a local version of a repository, including all commits and branches
remote: a common repository on GitHub that all team member use to exchange their changes
fork: a copy of a repository on GitHub owned by a different user
pull request: a place to compare and discuss the differences introduced on a branch with reviews, comments, integrated
tests, and more
HEAD: representing your current working directory, the HEAD pointer can be moved to different branches, tags, or commits
when using git checkout
**

  • 解释

git: an open source, distributed version-control system

git：一个开源的分布式版本控制系统

GitHub: a platform for hosting and collaborating on Git repositories

GitHub：托管和协作Git存储库的平台

commit: a Git object, a snapshot of your entire repository compressed into a SHA

提交：一个Git对象，将整个存储库的快照压缩到SHA中

branch: a lightweight movable pointer to a commit

分支：指向提交的轻量级可移动指针

clone: a local version of a repository, including all commits and branches

克隆：存储库的本地版本，包括所有提交和分支

remote: a common repository on GitHub that all team member use to exchange their changes

remote：GitHub上的一个公共存储库，所有团队成员都使用它来交换更改

fork: a copy of a repository on GitHub owned by a different user

fork：GitHub上由不同用户拥有的存储库的副本

pull request: a place to compare and discuss the differences introduced on a branch with reviews, comments, integrated
tests, and more

拉式请求：一个比较和讨论分支上引入的差异的地方，包括评论、评论、集成测试等等

HEAD: representing your current working directory, the HEAD pointer can be moved to different branches, tags, or commits
when using git checkout

HEAD：表示当前工作目录，HEAD指针可以移动到不同的分支、标记或提交使用git签出时
